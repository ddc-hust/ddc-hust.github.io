1. Git基本命令

  1. 工程区域：
    1. 版本库：
    2. 工作区
    3. 暂存区
  2. git pull命令：获取远端某个分支的更新，并合并到本地分支
    • git pull origin remote_branch : local_branch
    • git pull = git fetch + git merge
  3. git push命令：
    • git push origin local_branch : remote_branch
  4. git fetch命令：获取远端某个分支的更新到本地仓库，但是不执行合并操作，没有到工作区
    • git fetch origin remote_branch : local_branch
  5. git merge命令：合并指定分支内容到当前所在分支
    • git merge不会对历史提交节点进行破坏，而是生成一个新的合成节点（这样导致git log查看的时候不是很方便）
    • git merge branch_name
  6. git rebase命令：合并指定分支内容到当前所在分支。功能等同于git merge，但是背后原理不同
    • git rebase将被合并分支的节点融合到合并分支上，相当于被合并分支上所有的节点都要生成新的节点。
    • 注意：不要在主干分支上使用git rebase，这使得主干分支上可能出现两个修改内容相同，但是commit id不同的节点
    • git rebase barch_name
  7. git reset命令：撤销工作区中的一些git add/commit的操作。
    • git reset commit_id。（commit_id可通过git log查看）
  8. git checkout命令：回退本地所有修改但未提交的文件内容。谨慎使用
    • git checkout -filename：撤销该文件上工作区的修改，回退到原始版本
    • git checkout
    • git checkout commit_id
  9. git commit命令：提交修改到本地仓库
    • git commit --amend：修改最近一次提交记录的commit信息
    • git commit -am xxx：增加的a表示提交暂存区所有文件
  10. git add命令：提交修改到暂存区
    • git add .：提交所有修改
  11. git log命令：查看修改日志
    • git log --name-status：查看修改状态
    • git log -number：查看最近number条修改记录
  12. git diff命令：查看差异
    • git diff branch1 branch2：查看两个分支上的差异
    • git diff commit_1 commit_2：查看两个commit_id节点的差异
  13. git cherry-pick命令：去某一个提交节点内容到当前分支上，并生成新的提交节点id
    • git cherry-pick commit_id
  14. git reset命令：撤销某个操作，使分支返回到某个提交节点
    • git reset --hard commit_id：强制使分支回到commit_id节点，慎用，没有提示
  15. git reflog命令：查看git历史操作命令
  16. git remote命令：
    • git remote add origin xxx.git：本地仓库和远端仓库进行相连

2. 仓库

  1. fork仓库并创建merge request：->当要修改目标仓库的内容时，不用git clone修改，而是fork仓库后，在自己的仓库上修改完成后，再提交merge request

    • 可线上解决冲突
    • 也可克隆仓库到本地，在本地解决冲突
