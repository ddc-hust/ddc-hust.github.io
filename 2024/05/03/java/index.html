java编程入门

1. 基本数据类型及应用

  1. 数值型：
    • 整数类型：byte, int, short, long
    • 浮点型：float, double
  2. 字符型: char
  3. 布尔型: boolean

2. 引用数据类型及应用

  • 引用类型，在栈内存中存储地址，在堆内存中存储实际值

  1. 类

    • 包装类：对基本数据进行包装，需要注意的是，两个包装对象想比较的时候，比较的是内存中的引用地址。包装类默认为null

    • 常量池：java对简单数字（-128-127）对应的包装类进行了缓存。通过直接量赋值的包装类如果在该范围内的话，会直接使用常量池中的引用。

  2. 接口

  3. 数组：int[] arr = new int[5]

  4. 枚举

  5. 注解

3. java基本语法

  1. java程序

    • java程序由不同的包组成，包中包含不同的java文件，每个java文件只能由一个和文件名同名的公共类\接口

  2. 标识符和关键字

    • 公司命名规则：类名首字母大写，变量和方法名采用驼峰标志法，常量全大写，多个单词之间用_分割，包名全小写

  3. 异常处理

    • 每个catch子句依次检查，编写多重catch语句时，需要先小后大，即先子类再父类

    • finally执行语句不会影响原有的返回值（因为finally的执行时机在try/catch代码推出前，即所有代码块后，跳出逻辑前）

    • finally中的返回语句会代替原有的返回逻辑

  4. 匿名内部类——语法糖

    • 省略了内部类名

  5. Lamda表达式——语法糖

    • 省略接口名，函数名

    • Lamda只能作用于函数接口上，即只有一个方法的接口

    • Lamda依赖于参数推断

    • 语法格式：

    • 可选类型声明：不需要声明参数类型，编译器可以统一识别参数值

    • 可选的参数圆括号：一个参数无需圆括号，多个参数需要圆括号

    • 可选大括号：若主体只包含一个语句，就不需要使用大括号

    • 可选的返回关键字：若主体只有一个语句，则编译器会自动返回该表达式的值，无需return关键字

4. IO编程

  • 典型媒介：文件、管道、网络、内存缓存、java标准流（System.in/ System.out/System.error）
  1. 字节流：以字节为单位处理数据，适合处理二进制数据
    • InputStream
    • OutputStream
  2. 字符流：以unicode码元为单位处理数据，适合处理文本数据
    • Reader
    • Writer
  3. NIO：同步非阻塞型IO(IO多路复用) Non-blocking IO
    • Buffer(缓冲区)：缓冲区本质为一块可读写数据的内存。（Java NIO面向缓冲区，IO面向流）
    • Channel(通道)：1. 类似流，但有区别。2. 通道是双向的，可读可写。 3. 通道中的数据从Buffer中读取，或者从Buffer中写入
    • Selector(选择器)：用于侦听多个通道（channel）是否已做好读写准备，配合非阻塞IO实现单个线程管理多个通道
      • 多路复用的IO通过使用选择器去监听多个通道，使得效率提升
      • （PS：现代操作系统和多核cpu已越来越适应多任务处理，线程开销越来越小，IO多路复用已不是提高并发IO性能的手段）

5. 正则表达式

  • java中两个反斜杠才具有转移作用

  • 调用类

    • Pattern类：正则表达式的编译表示
    • Matcher类：对输入字符串进行解释和匹配操作的引擎
    • PatternSyntaxException：非强制异常类

  • 捕获组

    • group(0)是一个特殊的组，代表整个表达式

6.面向对象

1. 基本语法

  • 类：归纳和整理

    • class 类名 {	// 类名首字母大写
    特征（属性）;
    功能（方法）;
}

    • 对象是将内存地址赋给变量，变量是引用内存中的对象，称之为引用变量。变量的类型称之引用数据类型。

    • 空对象：没有引用的对象，取值为null。（所有引用类型变量的默认取值是null）

2.静态

  • 把和对象无关，只和类相关的称之为静态。（针对具体对象的属性，称为对象属性。针对具体对象的方法，称为对象方法。）

    • 静态属性：和类相关的属性，和对象无关

    • 静态方法：和类相关的方法，和对象无关

    • 语法：在属性和方法前加关键字static

    • 访问：直接通过类名即可访问静态方法和静态属性。（也可通过对象实例访问静态方法，有对象了必然有类）

    • class Bird {
    static String type = "鸟";	// 静态属性
    static void fly() {		// 静态方法
        System.out.prinln("飞。。。");
    }
}
System.out.prinln(Bird.type);	// 使用类名访问静态属性，这里的system的out方法也是
Bird.fly();		// 使用类名访问静态方法

4. 构造方法

  • 构造方法也是方法，但是没有void返回值

  • 方法名和类名完全相同

  • 如果类中没有构造方法，那么JVM会提供默认的构造方法

  • 构造方法也是方法，可以传递参数，一般传递参数目的是用于对象属性的赋值

  • 类名() {
}
创建对象：new 类名()其实就是调用构造方法

  • 创建子类对象时，会先创建该子类的父类对象（说法不准确，是会继承父类的属性和方法），即会先执行父类对象的构造方法.（代码层面：子类构造方法中会执行super函数代指父类的构造方法）

  • class Parent {
    Parent() {
        System.out.println("Parent...");
    }
}

class Children {
    Children() {
        super();	// 这个就是调用父类的构造函数，不写的话，JVM会自动补上
        System.out.println("Children");
    }
}

class Parent {
    String name;
    Parent(String name) {
        this.name = name;
        System.out.println("Parent..." );
    }
}

class Children {
    Children() {
        super("zhanagsan");	// 父类构造方法有参数，需要显示super函数，显示调用函数
        System.out.println("Children");
    }
}

7. 重载

  • 重载：同名方法在不同场景下表现不同
  • 相同方法：方法名、参数列表相同，和返回类型无关
  • 一个类中不能声明相同的方法，也不能声明相同的属性
  • 如果方法名相同，但是参数列表（个数、顺序、类型）不同，会认为是不同的方法，只不过名称一样。
  • 构造方法也可以重载

8. 重写

  • 父类对象的方法主要体现通用性，无法在特殊场合下使用
  • 如果子类对象需要在特殊场合下使用，那么需要重写方法的逻辑，
  • 重写：并不 意味着父类的方法被覆盖掉，只是在当前场合下不适用，如果使用super关键字，还是可以访问
  • 要求：子类的方法和父类的方法，方法名相同，返回类型相同，参数列表相同

9. 访问权限修饰符

  • java的源码中，公共类只能有一个，且必须和文件名相同

  • main方法由JVM调用，JVM调用时应该可以任意调用，而不需要考虑权限问题

  • public class Person {
    public static void main(String[] args) {}	// 这里public是为了JVM调用，这里的static是因为main方法是脱离类的，
}

10. 内部类

  • 外部类：在源码中直接声明的类

  • 内部类：在类中声明的类，内部类当成外部类的属性使用即可，所以使用内部类时需要构建外部类的对象

  • public class MainClass {
    public static void main(String[] args) {
        OuterClass outer =  new OuterClass();
        OuterClass.InnerClass innerClass = outer.new InnerClass();
    }
}

class OuterClass {
    pulic class InnerClass {
        
    }
}

  • final可以修饰方法，但这个方法不可以被子类重写

  • final可以修饰类，这样这个类就没有子类

  • final不可以修饰构造方法

  • final可以修饰方法的参数，一旦修饰，参数无法修改

13. 抽象

  • 抽象类：不完整的类就是抽象类，因为类不完整，所以无法直接构造对象

    • abstract class 类名

  • 如果一个类中含有抽象方法，这个类一定是抽象类

  • 如果一个类是抽象类，它的方法不一定是抽象方法

    • abstract class Person {
    public abstract void eat();	// 抽象方法没有实现体
}

14. 接口

  • 接口：即规则，接口是抽象的，

  • 规则的属性必须为固定值，而且不能修改。属性是静态的，行为应该是抽象的

  • 属性和行为的访问权限必须为公共的，

  • interface 接口名称{ 规则属性， 规则的行为}
