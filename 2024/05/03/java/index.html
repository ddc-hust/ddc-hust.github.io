java编程入门

1. 基本数据类型及应用

  1. 数值型：
    • 整数类型：byte, int, short, long
    • 浮点型：float, double
  2. 字符型: char
  3. 布尔型: boolean

2. 引用数据类型及应用

  • 引用类型，在栈内存中存储地址，在堆内存中存储实际值

  1. 类

    • 包装类：对基本数据进行包装，需要注意的是，两个包装对象想比较的时候，比较的是内存中的引用地址。包装类默认为null

    • 常量池：java对简单数字（-128-127）对应的包装类进行了缓存。通过直接量赋值的包装类如果在该范围内的话，会直接使用常量池中的引用。

  2. 接口

  3. 数组：int[] arr = new int[5]

  4. 枚举

  5. 注解

3. java基本语法

  1. java程序

    • java程序由不同的包组成，包中包含不同的java文件，每个java文件只能由一个和文件名同名的公共类\接口

  2. 标识符和关键字

    • 公司命名规则：类名首字母大写，变量和方法名采用驼峰标志法，常量全大写，多个单词之间用_分割，包名全小写

  3. 异常处理

    • 每个catch子句依次检查，编写多重catch语句时，需要先小后大，即先子类再父类

    • finally执行语句不会影响原有的返回值（因为finally的执行时机在try/catch代码推出前，即所有代码块后，跳出逻辑前）

    • finally中的返回语句会代替原有的返回逻辑

  4. 匿名内部类——语法糖

    • 省略了内部类名

  5. Lamda表达式——语法糖

    • 省略接口名，函数名

    • Lamda只能作用于函数接口上，即只有一个方法的接口

    • Lamda依赖于参数推断

    • 语法格式：

    • 可选类型声明：不需要声明参数类型，编译器可以统一识别参数值

    • 可选的参数圆括号：一个参数无需圆括号，多个参数需要圆括号

    • 可选大括号：若主体只包含一个语句，就不需要使用大括号

    • 可选的返回关键字：若主体只有一个语句，则编译器会自动返回该表达式的值，无需return关键字

4. IO编程

  • 典型媒介：文件、管道、网络、内存缓存、java标准流（System.in/ System.out/System.error）
  1. 字节流：以字节为单位处理数据，适合处理二进制数据
    • InputStream
    • OutputStream
  2. 字符流：以unicode码元为单位处理数据，适合处理文本数据
    • Reader
    • Writer
  3. NIO：同步非阻塞型IO(IO多路复用) Non-blocking IO
    • Buffer(缓冲区)：缓冲区本质为一块可读写数据的内存。（Java NIO面向缓冲区，IO面向流）
    • Channel(通道)：1. 类似流，但有区别。2. 通道是双向的，可读可写。 3. 通道中的数据从Buffer中读取，或者从Buffer中写入
    • Selector(选择器)：用于侦听多个通道（channel）是否已做好读写准备，配合非阻塞IO实现单个线程管理多个通道
      • 多路复用的IO通过使用选择器去监听多个通道，使得效率提升
      • （PS：现代操作系统和多核cpu已越来越适应多任务处理，线程开销越来越小，IO多路复用已不是提高并发IO性能的手段）

5. 正则表达式

  • java中两个反斜杠才具有转移作用

  • 调用类

    • Pattern类：正则表达式的编译表示
    • Matcher类：对输入字符串进行解释和匹配操作的引擎
    • PatternSyntaxException：非强制异常类

  • 捕获组

    • group(0)是一个特殊的组，代表整个表达式

6.面向对象

1. 基本语法

  • 类：归纳和整理

    • class 类名 {	// 类名首字母大写
    特征（属性）;
    功能（方法）;
}

    • 对象是将内存地址赋给变量，变量是引用内存中的对象，称之为引用变量。变量的类型称之引用数据类型。

    • 空对象：没有引用的对象，取值为null。（所有引用类型变量的默认取值是null）

2.静态

  • 把和对象无关，只和类相关的称之为静态。（针对具体对象的属性，称为对象属性。针对具体对象的方法，称为对象方法。）

    • 静态属性：和类相关的属性，和对象无关

    • 静态方法：和类相关的方法，和对象无关

    • 语法：在属性和方法前加关键字static

    • 访问：直接通过类名即可访问静态方法和静态属性。（也可通过对象实例访问静态方法，有对象了必然有类）

    • class Bird {
    static String type = "鸟";	// 静态属性
    static void fly() {		// 静态方法
        System.out.prinln("飞。。。");
    }
}
System.out.prinln(Bird.type);	// 使用类名访问静态属性，这里的system的out方法也是
Bird.fly();		// 使用类名访问静态方法

4. 构造方法

  • 构造方法也是方法，但是没有void返回值

  • 方法名和类名完全相同

  • 如果类中没有构造方法，那么JVM会提供默认的构造方法

  • 构造方法也是方法，可以传递参数，一般传递参数目的是用于对象属性的赋值

  • 类名() {
}
创建对象：new 类名()其实就是调用构造方法

  • 创建子类对象时，会先创建该子类的父类对象（说法不准确，是会继承父类的属性和方法），即会先执行父类对象的构造方法.（代码层面：子类构造方法中会执行super函数代指父类的构造方法）

  • class Parent {
    Parent() {
        System.out.println("Parent...");
    }
}

class Children {
    Children() {
        super();	// 这个就是调用父类的构造函数，不写的话，JVM会自动补上
        System.out.println("Children");
    }
}

class Parent {
    String name;
    Parent(String name) {
        this.name = name;
        System.out.println("Parent..." );
    }
}

class Children {
    Children() {
        super("zhanagsan");	// 父类构造方法有参数，需要显示super函数，显示调用函数
        System.out.println("Children");
    }
}

  • 多态可基于继承普通类、抽象类实现，也可基于接口实现（多个类实现同一个接口）

  • 多态要求向上转型，将子类对象赋值给父类引用。或者将实现类对象赋值给接口引用

  • 多态要求重写父类方法，或实现抽象类/接口的抽象方法

  • class Client {
    public void demo() {
        A a = new B();
        B b = new B();
        System.out.println(a.show(b)); // 输出B and A
    }
}

class A {
    public String show(D obj) {
        return "A and D";
    }
    public String show(A obj) {
        return "A and A";
    }
}
class B extends A {
    public String show(B obj) {
        return "B and B";
    }
    public String show(A obj) {
        return "B and A";
    }
}
class C extends A {}
class D extends B {}
// 首先a的类型是A, show方法会区A类中找，发现A类中没有接受参数为B的show方法，但是B是A的子类，所以show(A obj)方法可以接收，但是a的实现是B的对象，会再去B类中找show方法，所以输出B and A
// A a = new B();是超类对象引用变量a引用子类B的对象，最终由B对象的类型来决定调用谁的成员方法，但前提是这个方法必须在超类中定义过

  • private：私有的，同一个类中可以使用

  • default：默认权限，当不设定权限时，JVM会默认提供权限，包（路径）权限

  • protected：受保护的权限，子类可以访问

  • public：

  • package com;
public class User {
    String sex;
    private name;
    public age;
}

package com;
public class User2 {
    pulic static void main(String[] arga) {
        User u = new User();

11. final

  • final：在数据初始化后，不可被修改，被称为常量

  • final修饰变量，变量的值一旦初始化后无法修改

  • final修饰类的属性，JVM无法自动进行初始化，需要自己进行初始化，可在构造函数中初始化

  • class User {
    public final String name;
    public User(String name) {
        this.name = name;	// 不可以写name = name指代不明，需要使用this
    }
}

  • 抽象方法：只有声明，没有实现的方法

    • abstract 返回值类型 方法名（参数）

  • 抽象到具体：通过继承实现

    • 抽象类无法直接构建对象，但可通过子类间接构造对象

    • 如果抽象类中含有抽象方法，那么子类继承抽象类，需要重写抽象方法，

    • abstract关键字不可以和final同时使用，因为abstract抽象必须要有具体的实现

    • abstract class Person {
    public abstract void eat();
}

class Chinese extends Person {
    public void eat() {
        System.out.println("xxx");	// 具体的实现方法
    }
}

  • 接口与类：

    • 接口可以继承（extends）其他接口，不能说implements实现
    • 类的对象需要遵循接口，在java中，遵循表示实现，类需要实现接口，而且可以实现多个接口
  • interface USBInterface {
    
}

interface USBSupply extends USBInterface {
    public void powerSupply();
}

interface USBReceive extends USBInterface {
    public void powerReceive();
}

class Computer implements USBSupply {	// 类实现接口
    public USBReceive usb1;
    public USBReceive usb2;
    
    public void powerSupply() {
        System.out.println("电脑提供电源");
        usb1.powerReceive();		// 插入接口
        usb2.powerReveive();
    }
}

class Light implements USBReceiver {
    public void powerReveive() {		// 某个接口的具体实现方法，提供能源的具体方法
        System.out.println("电灯提供能源");
    }
}

Computer c = new Computer();
Light light = new Light();	// 创建两个具体的电灯实例
c.usb1 = light;
Light light2 = new Light();
c.usb2 = light2;
c.powerSupply();	// 两个电灯提供额能源


* 注意：

  * 不能对`boolean`类型进行转换
  * 浮点数转整数，是通过舍弃小数得到的，不是四舍五入
  * 把容量大的类型转换为容量小的类型，必须使用强制类型转换

### 3. 引用类型与拆装箱

* `jdk`为每个基本数据类型提供了一个包装类。（`char`类型的包装类是`Character`）

* 拆装箱会影响性能，尽量避免使用拆装箱

* **比较**：对对象进行比较使用`equals`方法，这样比较的就是值。如果使用等号的话。将比较的是引用的地址

* ```java
  Integer i1 = 100;
  Integer i2 = 100;
  System.out.prinln(i1 == i2); // 输出true，因为在Integer源码中，对i1和i2进行初始化时，使用的是缓存地址，所以二者指向同一个地址
  
  Integer i1 = 100;
  Integer i2 = new Integer(100); // 输出false
